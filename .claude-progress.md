# Generative UI MVP — Progress Log

## Goal
Add TeaNode-native structured render payloads (surfaces), a unified interrupt model,
a SurfaceRenderer in the web UI, and a user-action round-trip — a working end-to-end
vertical slice across Go backend + React frontend. No external AG-UI/CopilotKit deps,
schema-driven only.

## Architecture decisions
- Mirror the established `questions`/`approvals` integration pattern: a new
  `internal/integrations/surfaces` package with types + in-memory broker + context helper.
- New pubsub event `conversationSurfaces` carries a render payload `{action, surface?, interrupt?}`
  parallel to `conversationQuestions`/`conversationApprovals`. (This is the
  "conversation events carrying structured render payloads" requirement.)
- Emission paths (end-to-end testable, not throwaway):
  1. `render_surface` builtin tool — agent-driven, non-blocking emit.
  2. `surfaces.emit` debug RPC — push a surface/interrupt to a conversation without an LLM.
- Round-trip: `surfaces.action` RPC. MVP routes the action through the coordinator by
  injecting a synthetic user message (visible, resumes the run loop). Broker validates
  the surface/interrupt exists and clears resolved interrupts.
- Unified interrupt is primarily a frontend concept: questions, approvals, and new
  choice/form/review interrupts are adapted into one `Interrupt[]` model and rendered
  through a shared `InterruptRenderer` (existing QuestionPanel/ApprovalPanel reused
  under the hood). Backend defines the `Interrupt` struct so types exist on both sides.
- Surface locations: `inline` (rendered in the conversation column near latest content)
  and `right_panel` (a dedicated `SurfaceSidePanel`, sibling to the artifact panel — kept
  separate from artifact plumbing to limit blast radius).

## Surface schema (schemaVersion 1)
Surface: surfaceId, schemaVersion, location, title, components[].
Components (tagged by `type`): Section, Markdown, KeyValueList, Table, StatusBadge,
ButtonRow, Form, CodeBlock, Timeline.
Form fields: TextInput, Textarea, Select, Checkbox.
Interrupt kinds: question, approval, choice, form, review.

## Remaining issues / follow-ups
- `surfaces.action` currently resumes the runtime by injecting a synthetic user-visible message; a future iteration should add first-class surface action / interrupt resume semantics instead of text injection.
- Surface updates are MVP-level full-state emit/list semantics; there is no generalized patch protocol yet.
- The `render_surface` builtin tool currently emits surfaces only; emitting backend-native interrupts from tools can be added next.
